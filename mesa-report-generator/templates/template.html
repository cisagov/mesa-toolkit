MESA Results

CISA's Micro-Evaluation and Security Assessment (MESA) service enables CISA to drive risk reduction by delivering micro internal network security assessment services to those critical infrastructure and service providers in the greatest need. MESA is a cost-effective and scalable program that focuses on risk reduction by delivering valuable assessment services to less mature and under-resourced customers, collaborating with them to quickly develop a deeper, more risk-informed understanding of their internal cybersecurity posture.

The results of the assessment are summarized below. Additional information can be found in the "Data" folder, accessible via the Quick Access dropdown above. Each top-level folder represents one of the services conducted by the Cybersecurity and Infrastructure Security Agency (CISA) team, and within those folders, sub-folders containing raw data from the tools leveraged during the assessment can be located.

Results Summary

The following services were performed by the Cybersecurity and Infrastructure Security Agency (CISA) Team during the Micro Evaluation Security Assessment (MESA).

  • Host Discovery Scans

    CISA performed an automated discovery scan of the targeted environment. These scans were intended to identify the number of live hosts that existed within the provided scope.

    Scan Results:
    {{ ip_addr_scanned }} IP addresses were scanned
    {{ live_hosts }} hosts were identified as live on the network

    The raw results from the discovery scanning activity can be located within the Port_Scans/DISCOVERY folder.

  • Full Port Scans

    CISA performed an automated full port scan of the targeted environment. These scans were intended to identify the various services running on open ports. These results were based on the live hosts that were discovered on the network.

    Scan Results:
    {{ live_hosts }} hosts were scanned
    {{ unique_ports }} unique ports were identified as open across all live hosts

    The raw results from the full port scanning activity can be located within the Port_Scans/FULL folder.

  • Web Application Enumeration

    CISA performed an aqutone web application enumeration scan against the provided scope. This scan leveraged the results from the discovery scan to identify any existing web servers within the internal network.

    Scan Results:
    {{ web_servers }} web servers were scanned

    The results from the web application enumeration activity can be located within the Web_App_Enumeration folder. To view the report simply double click on the aquatone_report.html file located within the folder.

  • Encryption Check

    CISA performed an automated discovery scan using nmap of the targeted environment. The scan was specifically looking for hosts leveraging cleartext protocols within the scope provide. Each affected host has been aggregated into a list based on the discovered protocol ports (i.e 21-TCP.txt, 23-TCP.txt 80-TCP.txt, 8000-TCP.txt, 8080-TCP.txt). The raw results from the scan can be located in the Cleartext_Protocols folder.

    Scan Results:
    {{ live_hosts }} IP addreses were scanned
    {{ cleartext_hosts }} hosts were identified as leveraging cleartext protocols

    Additonally, CISA performed an SSL scan against discovered internal web applications leveraging the HTTPS protocol. The results from the scan, if any, can be found within the SSL_TLS_Encryption_Cipher folder. These results will show which SSL/TLC protocols are enabled, whether the targeted host is leveraging various common security configurations, and the various encryption ciphers being leveraged by the targeted servers. The targeted systems can be found in the Scan_Targets.txt file for ease of reference.

  • Default Logins

    CISA performed a discovery scan to identify the exisitence of common protocols that leverage login mechanisms. The scan performed specifically checked for the FTP, TELNET, HTTP, and HTTPS protocols (ports 21, 23, 80, 443, 8000, 8080, 8443). Nuclei templates (https://github.com/projectdiscovery/nuclei-templates/tree/main/http/default-logins) were leveraged to check for the existance of default username password combinations that could be used to authenticate to the targeted services.

    Scan Results:
    {{ default_logins }} instances of various forms of default logins were identified on the network

    The results from the default login scanning activity can be located within the Insecure_Default_Configuration/Default_Logins folder. The affected hosts can be identified by viewing the corresponding text files with "affected_hosts.txt" appended to them.

  • Vulnerability Scans

    CISA performed a vulnerability scan using nuclei, a template based vulnerability scanner (https://github.com/projectdiscovery/nuclei.git). The results include JSON files (RV1234_[severity]_findings.json) that contain all of the information associated with the template url, high level and detailed description of the identified vulnerability, affected host, and much more. Additionally, the results will contain a summarized text file (RV1234_[severity]_affected_hosts.txt) containing the severity rating, high level finding description, and the affected hosts.

    {{ critical_vulns }} critical vulerabilities identified

    {{ medium_vulns }} medium vulnerabilities identified
    {{ low_vulns }} low vulnerabilities identified

    Please note that this summary represents a snapshot in time and further analysis should be conducted periodically to ensure additional vulnerabilities haven't been introduced into the environment.

  • SMB Signing Checks

    CISA performed a check for workstations and servers not currently leveraging SMB signing. SMB (Server Message Block) signing is a security feature used in Microsoft Windows and some other operating systems to protect against certain types of network-based attacks and ensure the integrity and authenticity of data exchanged between networked devices. SMB is a network protocol used for sharing files, printers, and other resources between computers on a network.

    SMB signing works by adding a digital signature to each SMB packet sent between devices. This digital signature is created using a shared secret key and cryptographic algorithms, and it verifies that the data within the packet has not been tampered with during transit and that it originated from a trusted source. When an SMB packet is received, the recipient can check the digital signature to ensure its validity. Enabling SMB signing can protect against data tampering, adversary-in-the-middle attacks, and unauthorized access.

    Scan Results:
    {{ smb_sign_disabled }} of {{ live_hosts }} scanned IP addresses did not have SMB signing enabled
